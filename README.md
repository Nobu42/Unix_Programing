# Unix/Linux システムプログラミング

名著『Unix/Linuxプログラミング理論と実践』を通した、OS内部構造の検証記録リポジトリです。
「OSの裏側で何が起きているのか」を、C言語のソースコードとシステムコールを通して探求しています。

**※ 読了・完遂まで毎日更新中**

## 学習リソースとガイドライン
- **[DEVELOPMENT_GUIDE.md](./DEVELOPMENT_GUIDE.md )**：【重要】標準コマンド再実装のための実装プロセス・ガイド
- **[UNDERSTANDING_OS.md](./UNDERSTANDING_OS.md)**：OSの役割、リソース管理、Unix哲学に関する考察まとめ
- **[ENVIRONMENT.md](./ENVIRONMENT.md)**：コンパイル環境および検証用OSの構成詳細

## 本プロジェクトの取り組み

- **実装プロセスの標準化**
  `Memo.md` を道標とし、`man`読解から構造体特定、システムコール選定に至るまでの汎用的な実装フローを確立。
- **理論の言語化と検証**
  `UNDERSTANDING_OS.md` にてOSの基本原則を整理し、自作コードによる実証を繰り返しています。
- **ビルド環境と共通化**
  各ディレクトリに `Makefile` を配備。また、`common/` ライブラリにてユーザー空間バッファリング等の共通処理を実装し、システムコールの最適化を追求。
- **低レイヤーからの機能再現**
  iノード、stat構造体、ビットマスク、再帰アルゴリズム等を用い、標準コマンドの挙動を基底層から再現。

## ディレクトリ構造

- **who/** : ログイン管理とバッファリング処理
- **ls/** : stat構造体とビットマスクによる属性解析
- **pwd/** : iノードと再帰によるパス解決
- **more/** : 端末ページャの基本実装
- **cp/** : ファイルコピーとエラーハンドリング
- **chap02ex/**: 第2章 演習問題（cat, tail等の再実装）
- **common/** : プロジェクト共通ライブラリ（utmplib等）
- **shakyo/** : 写経によるコードリーディング記録
