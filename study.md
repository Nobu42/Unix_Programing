# Unix/Linux システムプログラミング：学習の記録とロードマップ

このファイルでは、名著『Unix/Linuxプログラミング理論と実践』を通じ、OSの深淵を探求する過程を記録します。

---

## 1. 探求テーマ（現在位置）
「プログラム」と「OS（カーネル）」の境界線はどこにあるのか？以下の仕組みを解明することを目標としています。

- **プログラムの実行**: バイナリがメモリに載り、プロセスとして動き出すまで。
- **ファイル管理**: ディスク上のデータがどのように抽象化され、操作されるのか。
- **データ処理の真実**: 入出力（I/O）、送受信、通信の背後で起きていること。
- **役割分担**: 「プログラムは何をなし、OSは何を助けているのか」の明確化。

## 2. これからのロードマップ（Unixシステムの主要構造）
Unixの設計思想を理解するための、自分なりのチェックリストです。

### 空間と接続の概念
- **メモリの分離**: 「ユーザー空間」と「システム空間（カーネル）」の厳格な切り分け。
- **カーネルの独占**: システム空間は、カーネルとそのデータ構造によってのみ占められている。
- **プロセスの居場所**: ユーザープロセスは常にユーザー空間に存在し、特権操作はカーネルに依頼する。

### デバイスとネットワーク
- **端末（TTY）**: ユーザーとシステムの接点。物理的な配線からカーネルへの接続まで。
- **ファイルシステム**: ディスク上のデータを構造化して保持する仕組み。
- **デバイスの抽象化**: あらゆるデバイスを「ファイル」として操作できるUnixの凄み。
- **ネットワーク接続**: 外部からシステムへ繋がるための入り口と、その制御。

## 4. 実装解説（Hands-on Implementation）
各ディレクトリに収められているコードの解説と、学習した理論の紐付けです。

### more / who / cp
- **各プログラムの役割**: コマンドラインツールの基本実装。
- **utmplib (Buffering)**:
  - **学び**: システムコールを最小限にし、ユーザー空間にバッファを設けることで劇的に高速化する仕組み。
  - **実践**: `who3` での実装を通じ、ディスクI/Oとメモリ操作の速度差を実感。

### ログアウトと後処理（logout_tty）
- **状態の遷移**: `utmp` レコードの `ut_type` を `DEAD_PROCESS` に書き換えることでログアウト状態を表現する。
- **ファイル操作**: `lseek` を用いて、直前に `read` したレコードの位置までファイルポインタを戻し、正確に上書き更新（In-place update）を行う。
- **整合性**: `utmp` や `wtmp` ファイルへの記録タイミングと、システムリソース解放の関連性。
