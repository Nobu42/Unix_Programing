# 演習課題設計指針：tailコマンドの実装

## 1. コマンド仕様と振る舞い (Manual)
- **基本動作**: ファイルの末尾から指定した行数（デフォルト10行）を表示する。
- **調査対象**:
    - `man tail` でオプション（`-n` など）を確認。
    - ファイル末尾が改行で終わっていない場合の挙動を確認する。
    - `strace tail [file]` を実行し、本物の `tail` が `lseek` をどう使っているか、一度に何バイト読み込んでいるか（バッファサイズ）を観察する。

## 2. データの特定と構造 (Search)
- **参照先**: 引数で指定されたファイル。
- **データ構造**: 行単位のテキストデータ。
- **実装の鍵**: 「ファイルの末尾」から「目的の行の開始位置」まで、バイナリとして遡る。

## 3. 使用する武器 (Syscalls)
標準入出力（stdio.h）はバッファリングにより逆方向の操作が複雑になるため、システムコールを直接使用する。

- `open()`: ファイルを読み取り専用で開く。
- `lseek()`:
    - `lseek(fd, 0, SEEK_END)` で末尾へ。
    - `lseek(fd, -offset, SEEK_CUR)` で現在位置から手前へ戻る。
- `read()`: バッファにデータを読み込み、`\n` をスキャンする。
- `write()`: 特定した位置から末尾までを標準出力（fd: 1）へ書き出す。

## 4. 基本設計アルゴリズム (Design)

### ステップ1：末尾への到達
1. `open` でファイルを開く。
2. `lseek` を使い、ファイルポインタを末尾（SEEK_END）に移動させる。

### ステップ2：逆方向スキャン
1. 末尾から手前方向へ一定量（または1バイトずつ）ポインタを戻す。
2. `read` で読み込み、内容が改行コード `\n` であるか判定する。
3. 指定の行数（10行なら10個）の改行が見つかるまで繰り返す。
   - ※ファイルの先頭（オフセット0）に到達した場合は、そこでスキャンを終了する。

### ステップ3：出力
1. 目的の行数が見つかった直後の位置（オフセット）を保持する。
2. `lseek` でその位置まで移動（SEEK_SET）する。
3. そこからファイル末尾までを通常通り `read` し、標準出力へ `write` する。

## 5. 実装の注意点
- **オフセット管理**: `lseek` で戻りすぎるとエラーになるため、現在の位置をつねに把握しておく。
- **効率化**: 1バイトずつの `read` はシステムコールが多発し低速になる。1024バイト程度のバッファに読み込み、メモリ上で後ろからスキャンするのが望ましい。
- **例外処理**: ファイルサイズが指定行数より少ない場合、ファイル全体を表示して終了するように設計する。
