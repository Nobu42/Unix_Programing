## 第1章：Unix システムプログラミングの全体像
	#### システムプログラミングの基礎とOSの役割

### 1. オペレーティングシステム（OS）の本質
OSとは、コンピュータの全リソースを管理・保護し、多様なプログラムを統合・制御する存在である。

- **サービスの提供**: プログラムに対し実行環境（リソース）を提供する。
- **空間の分離**:
  - **カーネル空間（System Space）**: OS（カーネル）が独占し、特権操作を行う空間。
  - **ユーザー空間（User Space）**: ユーザープログラムが動作する制限された空間。
- **一元管理**: カーネルを介することで、リソースへのアクセスを正しく、効率的に、そして安全に保つ。

### 2. カーネルが管理する主要リソース

| リソース | カーネルの役割 |
| :--- | :--- |
| **プロセッサ** | プログラムへの実行権割当。開始、一時停止、再開、終了の制御。 |
| **入出力（I/O）** | 全データのゲートウェイとなり、一元管理による安全性と効率性を確保。 |
| **プロセス管理** | プロセスの生成、スケジューリング、リソース分配による協調動作の実現。 |
| **メモリ** | メモリ空間の割当と管理。他プロセスによるデータの破壊を防ぐ保護機能。 |
| **デバイス** | 多様なハードウェアへのアクセス提供と、複雑な通信プロトコルの隠蔽。 |
| **タイマー** | プロセスに対する高精度な時間計測機能の提供。 |
| **IPC / ネットワーク** | プロセス間通信（IPC）およびその拡張であるネットワークアクセスの提供。 |

### 3. システムプログラミングを理解するための3ステップ
単なる知識に留めず、以下のサイクルで技術を習得する。
1. **それは何をするのか**（機能の把握）
2. **どんな仕組みなのか**（内部構造の解明）
3. **自分でも試せるのか**（コードによる実装・検証）

## 第2章：ユーザー、ファイル、マニュアル-最初はwhoから
#### ファイル処理とシステム情報の取得

### イントロダクション
- **`who` コマンドの解析**:
  Unixにおける「ファイル」の取り扱い方を深く学ぶ。システムコールを用いてデータを読み解く過程を通じ、Unixの設計思想そのものを理解する。
- **Unixから学ぶ方法**:
  マニュアル（man）やソースコード、既存のツールをどのように活用して未知の技術を習得すべきか、その「学びの作法」を体得する。

### whoは何をするのか
- **マニュアル（man）を読む**
  コマンドの動作を確認する第一歩。どのシステムファイルを参照しているかを特定する。

### whoはどんな仕組みなのか
- **ファイルから構造体を読み出すにはどうすれば良いか**
  - `open`, `read`, `close` のシステムコールを組み合わせ、構造体単位でデータを取得する。

### 私もwhoを書くことができるか
- 自身の環境でコンパイル・実行し、期待通りの出力が得られるかを検証する。

### プロジェクト２：cp を書く
- **cpは何をするのか**：ファイルの複製。既存のファイルを読み、新しいファイルへ書き出す。
- **cpはどんな仕組みか**：`creat` システムコールの役割と、パーミッションの設定。
- **自分にもcpを書くことはできるか**：エラーハンドリング（ファイルが存在しない、権限がない等）を含めた実装。

### もっと効率の良いファイル入出力：バッファリング
- **バッファのサイズに意味はあるのか**：ディスクI/Oの特性と、効率的なブロックサイズの検討。
- **システムコールに時間がかかるのはなぜか**：ユーザー空間からカーネル空間への「コンテキストスイッチ」のコストを理解する。
- **who2.cにバッファリングを追加する**：システムコールの回数を最小化する実装への改善。

### バッファリングとカーネル
- カーネル内部のバッファキャッシュと、ユーザー空間でのバッファリングの違い。

### ファイルの読み書き
- **ログアウト：何をしているのか**：`utmp` ファイルの書き換え処理。
- **ログアウト：どんな仕組みなのか**：特定のユーザーエントリを検索し、状態を更新する。
- **現在位置の移動：`lseek`**：ファイル内の特定位置へジャンプし、ランダムアクセスを実現する。
- **端末からログアウトするためのコード**：実装を通じた実証。

### システムコールのエラーはどのように処置したら良いのか
- `perror` や `errno` を用いた、堅牢なプログラムのためのエラー通知と対処法。
