# Linuxシステムプログラミング：実装プロセス・ガイド

新しいコマンド（tail, cat, lstat等）を実装する際や、課題に取り組む際の汎用的なステップ。

## 1. コマンドの「振る舞い」を定義する
まずは対象コマンドの正確な挙動を知ることから始める。

- `man [command]` : 基本的な使い方、オプション、動作仕様を確認。
- `man -k [keyword]` : 関連するシステムコールやファイル形式を逆引きする。
- 調査対象：
    - デフォルトで読み書きするファイルは何か？
    - 標準入力（stdin）とファイル指定の両方に対応しているか？
    - 特殊なエラーハンドリング（ファイルが存在しない場合など）はどうなっているか？

## 2. データの「出処」と「型」を特定する
プログラムがどこから情報を取得し、どのようなデータ構造で保持しているかを特定する。

- **ファイル・場所の特定：** - 設定ファイルやログファイル（`/etc/passwd`, `/var/run/utmp` 等）を読み取るのか。
    - カーネルから直接情報を得るのか（`stat` 構造体など）。
- **構造体の調査：**
    - `man 5 [filename]` または `man 2 [syscall]` でデータ構造を確認。
    - ヘッダーファイルの確認：`/usr/include/` 配下の該当ヘッダー（`utmp.h`, `stat.h`, `fcntl.h` 等）を `grep` や `vim` で覗き、メンバ変数を確認する。

## 3. 使用するシステムコールを選定する
C言語の標準ライブラリ（stdio.h）ではなく、より低レイヤなシステムコールを選択する。

- **基本の4要素：** `open()`, `read()`, `write()`, `close()`
- **メタデータ取得：** `stat()`, `lstat()`, `fstat()`
- **ファイルオフセット操作：** `lseek()`（`tail` や `od` の実装に必須）
- **ディレクトリ操作：** `opendir()`, `readdir()`

## 4. プログラムの基本を設計する
実装前に、データの流れをフローチャート化する。

1. **初期化：** 引数のパース（`argc`, `argv`）、ファイルのオープン。
2. **ループ処理：** - 「バッファサイズ分だけ読み込む」→「処理（整形）」→「書き出す」の繰り返し。
    - `read()` の戻り値が 0（EOF）になるまでループ。
3. **後処理：** ファイルのクローズ、メモリの解放、エラーステータスの返却。
